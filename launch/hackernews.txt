NirmiqLearn OS is a local-first learning OS that connects to Claude Code, Cursor, and Windsurf via MCP. The idea: if you're building with AI assistants, you should still be able to explain what you shipped.

It gives your AI assistant 10 MCP tools:

- Log debug sessions with structured root cause (error, suspected cause, actual cause, fix, lesson learned, prevention rule)
- Add explain-back questions to any workspace
- Link DSA/CS concepts to features you built
- Generate questions, identify concepts, analyse errors (AI-powered, BYOK — requires NIRMIQ_PRO_KEY + ANTHROPIC_API_KEY)
- Track daily logs and surface weak questions over time

The whole thing runs on your machine — Next.js 16 app, SQLite via Drizzle ORM, MCP server over stdio. Zero telemetry, zero cloud, zero network calls by default.

GitHub: https://github.com/SheeshDarth/NirmiqLearnOS
MIT licence.

The immediate use case is engineering students who build with Cursor/Claude Code for assignments and then struggle to explain the code in vivas or interviews. But it's useful for any developer who wants to build a habit of understanding, not just shipping.

Happy to discuss the MCP architecture, the local-first storage model, or the BYOK license design.
